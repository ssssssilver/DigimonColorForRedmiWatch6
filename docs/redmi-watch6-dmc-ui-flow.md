# REDMI Watch 6 DMC Ver.5 UI Flow

版本：0.1  
日期：2026-06-03  
职责边界：本文只定义 UI 原型、交互层级、画面状态、动效节奏和后续界面优化方向；不定义玩法公式、进化数据、战斗数值或存档实现。

## 1. UI 设计原则

### 1.1 首页优先是宠物待机画面

默认首页不是菜单页，也不是信息面板。默认状态应像一台正在运行的 Digital Monster COLOR Ver.5：

- 主视觉是宠物 sprite 在 LCD 场景中待机。
- 操作按钮默认隐藏。
- 状态信息只保留能影响照顾判断的最低限度。
- 用户点按屏幕后才唤出命令菜单。
- 菜单短暂停留，完成操作或返回后回到待机画面。

### 1.2 小屏单任务

REDMI Watch 6 目标画布为 432 x 514。每一屏只完成一个动作：

- 看宠物：待机首页。
- 选择动作：命令菜单。
- 执行动作：动作专属短演出。
- 看结果：1-2 秒结果反馈，点按可跳过。

避免在一个屏幕里同时放宠物、长状态、长消息和完整 9 宫格按钮。

### 1.3 原版感优先

视觉关键词：

- 黑底/暗绿 LCD。
- 硬边像素 sprite。
- 低帧动画。
- 短标签按钮。
- 高对比警告点缀。
- 数值用短条、心数、缩写表达。

避免：

- 现代手游大卡片。
- 长说明文案。
- 装饰性渐变。
- 过圆按钮。
- 流畅弹性动效。
- 大面积信息仪表盘。

## 2. 全局交互模型

### 2.1 默认态

默认态只显示：

- 顶部：`Ver.5`、当前宠物名、`Qx-y` / 年龄。
- 中部：LCD 场景、宠物、排泄物/伤病/CALL 等状态标识。
- 底部：饥饿与力量心。
- 最底：很弱的 `TAP SCREEN` 提示。

默认态不显示命令按钮。

### 2.2 唤出菜单

触发：

- 点按 LCD 或页面空白处。

结果：

- 底部出现 `COMMAND` 菜单。
- 主画面仍保留宠物，不跳到纯菜单页。
- 若有 CALL/排泄/伤病，菜单打开时显示更明确提示。

建议超时：

- 5-8 秒无操作自动收起菜单，回到待机首页。
- 执行动作后自动回到待机首页，FOOD 子菜单除外。

### 2.3 返回

返回优先级：

1. 子菜单返回命令菜单。
2. 命令菜单返回首页。
3. 演出/结果页点按跳过并回首页。
4. 危险操作如 RESET 进入确认屏，不直接执行。

## 3. 一级界面清单

### 3.1 Home / Idle

用途：长期停留的核心画面。

默认显示：

- 宠物 sprite。
- 简化场景。
- 饥饿心、力量心。
- 少量状态旗标：`CALL`、`SICK`、`HURT`、`ZZZ`、`COLD`。

待机动画：

- 2 帧或 3 帧低帧循环。
- 宠物轻微左右移动或上下跳动。
- 频率约 0.8-1.5 秒一拍。
- 不做丝滑 tween，不做现代呼吸光效。

状态叠加：

- 有排泄物：宠物旁边或地面显示小型像素物。
- 生病/受伤：LCD 顶部或宠物旁显示 `SICK/HURT`，可配短闪烁。
- 呼叫：`CALL` 标识短闪烁，前景可触发振动。
- 睡眠：宠物静止，显示 `ZZZ`，场景变暗。
- Cold Mode：宠物静止或冰冻色调，显示 `COLD`。

### 3.2 Command Menu

用途：从首页唤出的命令层。

入口：点按首页。

按钮：

- `FOOD`
- `TRAIN`
- `BATTLE`
- `WC`
- `MED`
- `LIGHT`
- `COLD`
- `STATUS`
- `DATA`
- `BACK`

布局建议：

- 2 行或 3 行短标签按钮。
- 菜单占底部 80-120 px。
- 不遮挡宠物主体。
- 当前有强提醒时，把最相关动作高亮，例如排泄时高亮 `WC`，生病时高亮 `MED`。

### 3.3 Status

用途：快速查看数值，不做百科页。

入口：

- 命令菜单 `STATUS`。

信息：

- 名称、阶段、年龄。
- 体重。
- 饥饿、力量。
- Effort。
- Care Mistake。
- 战斗胜负与胜率。
- Quest 进度。
- 进化倒计时。

布局：

- 单屏 6-8 行。
- 每行短缩写，不使用表格大列。
- 允许 `DATA` 进入更详细数据页。
- `RESET` 必须进入二次确认。

### 3.4 Food

入口：

- 命令菜单 `FOOD`。

子菜单：

- `MEAT`
- `VIT`
- `BACK`

交互：

- 点 `MEAT` 或 `VIT` 立即投喂。
- 投喂后留在 FOOD 子菜单，方便连续补心。
- 满心继续投喂时显示过量提示。

动画：

- 食物图标出现 1 秒以内。
- 宠物前移或跳一下。
- 结果文案如 `MEAT +1`、`FULL`、`OVER`。

禁止：

- 不进入长说明。
- 不使用现代弹窗卡片。

### 3.5 Train

入口：

- 命令菜单 `TRAIN`。

训练 UI：

- 一屏短小游戏。
- 中央显示力量槽或目标区。
- 用户点按停止。
- 结果为 `OK` / `MISS` / `EFF +`。

节奏：

- 从点击到结果目标 3 秒内。
- 结果页 1 秒左右，点按可跳过。

视觉：

- 像素条、硬边框、短文字。
- 宠物训练动作或撞击/闪烁演出。

### 3.6 Battle / Quest

入口：

- 命令菜单 `BATTLE`。

流程：

1. Battle Preview：敌人 sprite、名称、属性、Power、区域回合。
2. Confirm：点按开始，返回取消。
3. Battle Loop：我方/敌方交替 HIT/MISS。
4. Result：WIN/LOSE、Quest 推进、伤病提示。

画面原则：

- 不让用户在小屏翻 Quest 大表。
- 默认挑战当前 Quest 敌人。
- 战斗短促，接近原设备节奏。
- 结果 1-2 秒，点按可跳过。

### 3.7 WC

入口：

- 命令菜单 `WC`。

交互：

- 立即清理排泄物。
- 若没有排泄物，显示 `CLEAN`。

动画：

- 排泄物闪烁消失。
- 宠物轻跳或回到待机。

### 3.8 MED

入口：

- 命令菜单 `MED`。

交互：

- 立即治疗 `SICK/HURT`。
- 若无异常，显示 `OK` 或 `NO PROB`。

动画：

- 短闪烁。
- 状态旗标消失。

### 3.9 Light / Sleep

入口：

- 命令菜单 `LIGHT`。

状态：

- 睡眠时显示 `ZZZ`。
- 若用户唤醒，显示 `WAKE` 或 `LIGHT ON`。

视觉：

- 场景变暗。
- 宠物静止。
- 菜单只保留 `WAKE`、`STATUS`、`BACK` 等必要动作。

### 3.10 Cold Mode

入口：

- 命令菜单 `COLD` 或首页强入口。

状态画面：

- 显示 `COLD`。
- 宠物静止或低亮。
- 数值区可显示暂停标记。

交互：

- 冷冻中点击屏幕只出现 `RESUME`、`STATUS`、`BACK`。
- 不显示 FOOD/TRAIN/BATTLE 等不可用动作，避免误导。

### 3.11 Data

入口：

- 命令菜单 `DATA`。

用途：

- 展示简化记录，不做百科浏览器。

内容：

- Care Mistake。
- Effort。
- Overfeed。
- Battle count。
- Win rate。
- Quest clear。
- 当前版本固定 `Ver.5`。

后续可扩展：

- Quest 区域选择。
- 简化图鉴。
- 训练/进化记录。

限制：

- 不渲染长列表。
- 不在首版首页塞图鉴。

## 4. 特殊生命周期界面

### 4.1 Digitama / Hatch

显示：

- Digitama sprite。
- 简短倒计时或 `WAIT`。

动画：

- 蛋轻微晃动。
- 孵化时短闪白/绿。

操作：

- 首页点按可唤出少量信息。
- 不提供无意义操作。

### 4.2 Evolution

触发：

- 到达进化时间。

画面：

- 旧 sprite 闪烁。
- 中央显示 `EVOLUTION`。
- 新 sprite 出现。

节奏：

- 2-4 秒。
- 点按可跳过到结果。

结果：

- `NEW FORM` + 名称。
- 返回首页。

### 4.3 Death / Rebirth

显示：

- 墓碑或数据化符号。
- 暗色 LCD。
- 简短文案 `RETURNED TO DATA`。

操作：

- 只显示 `NEW EGG`。
- 点按后进入确认。

禁止：

- 不在死亡页显示普通菜单。

### 4.4 Save Recovery / Time Jump

用途：

- 存档异常或系统时间大幅跳变时。

显示：

- `SAVE ERR` / `TIME?`
- `CONTINUE`、`NEW EGG`。

视觉：

- 系统级异常页，但仍保持 LCD/像素样式。

## 5. UI 状态优先级

当多个状态同时存在时，首页显示优先级：

1. Dead
2. Evolution
3. Battle result
4. Cold
5. Sleep
6. Sick / Hurt
7. Call
8. Poop
9. Normal idle

菜单高亮优先级：

1. Dead -> `NEW EGG`
2. Cold -> `RESUME`
3. Sick/Hurt -> `MED`
4. Poop -> `WC`
5. Hunger/Power empty -> `FOOD`
6. Sleep -> `LIGHT`
7. Normal -> no highlight

## 6. 动画规范

### 6.1 首页待机

- 2-3 帧。
- 每帧停留 500-800 ms。
- 允许左右移动 6-14 px。
- 允许上下移动 3-6 px。
- sprite 必须保持硬边，不做模糊缩放。

### 6.2 呼叫

- `CALL` 以 1 秒节奏闪烁。
- 可短振动。
- 不弹现代通知。

### 6.3 操作反馈

- 投喂、清理、治疗、训练结果都控制在 1 秒左右。
- Battle/Evolution 可以更长，但不超过 4 秒。
- 所有演出允许点按跳过。

## 7. 原型推进顺序

### P0：主页与菜单基础

- Home idle。
- 点屏唤出 Command。
- 待机动画。
- 状态旗标。
- 菜单自动收起。

### P1：照顾动作

- Food 子菜单与投喂反馈。
- WC 清理反馈。
- MED 治疗反馈。
- Light/Sleep 状态。
- Cold Mode 状态。

### P2：训练与战斗

- Train 小游戏原型。
- Battle Preview。
- Battle Loop。
- Battle Result。

### P3：生命周期演出

- Digitama/Hatch。
- Evolution。
- Death/New Egg。
- Reset confirm。

### P4：数据与辅助页

- Status 精简页。
- Data 记录页。
- Quest 进度页。
- 后续简化图鉴/手册。

## 8. 当前 HTML 原型下一步

当前预览页应继续往这个方向推进：

1. 默认只显示待机画面。
2. 点屏进入 `COMMAND`。
3. 补齐 FOOD、TRAIN、BATTLE、COLD、STATUS 的真实视觉状态。
4. 每个动作都有短演出，不只是切文本。
5. 用同一套视觉规则再迁移到 Vela `.ux`。

## 9. 当前 UI 决策

- 首页不常驻 9 宫格。
- 菜单不是独立大页面，而是从首页唤出的底部命令层。
- 宠物待机必须有低帧动画。
- Cold/Sleep/Dead 等状态会改变可用菜单，而不是只弹提示。
- Status/Data 是辅助页，不抢首页主视觉。
- 首版只表现 Ver.5，不出现版本切换。
