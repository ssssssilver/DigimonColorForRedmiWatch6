# REDMI Watch 6 DMC 首页 UI 简化返工说明 v2

日期：2026-06-03  
用途：给开发落地首页 UI 与后续原型优化使用  
职责边界：本文只谈 UI、交互、视觉层级、动效与状态呈现，不涉及玩法规则、数值公式、进化数据或存档实现。

## 1. 当前截图的问题

当前实现方向已经接近，但仍然像“带宠物区域的功能面板”，还不像“手表里的宠物机待机画面”。主要问题：

- 首页仍有常驻 `MENU` 按钮。默认主页应没有显式按钮，点屏幕才出现菜单。
- 顶部品牌区、`A` 键、`DIGITAL MONSTER`、LCD 状态条和大边框形成多层装饰，信息密度太高。
- LCD 舞台被边框、状态条、心数盒子和底部按钮压缩，宠物显得太小。
- `I EVO 6m` 独占一整条横栏，视觉权重过高。
- 场景中的方块 skyline 比宠物更抢眼，当前不需要。
- `HUN` / `POW` 盒子太像现代仪表组件，可以更像原设备的短状态读数。
- 底部大面积空白只服务于 `MENU`，应还给宠物舞台或菜单浮层。

## 2. 首页目标

打开应用第一眼只应该看到：

- 一只宠物在 LCD 场景中待机。
- 顶部极小状态读数。
- 底部极小心数读数。
- 非强提示状态下没有按钮。

用户操作模型：

- 默认：看宠物。
- 点按屏幕：唤出命令菜单。
- 选择命令：进入动作短演出或子菜单。
- 返回/完成/超时：回到待机首页。

首页不应像菜单页。首页是宠物生活的画面。

## 3. 默认首页布局

目标画布：432 x 514。

推荐结构：

```text
0-26      顶部微型 HUD
26-430    宠物 LCD 舞台
430-466   心数/状态短读数
466-514   空白触摸区或弱 TAP 提示
```

注意：这只是分区，不代表要画出明显分割线。默认首页应尽量少线条。

### 3.1 顶部微型 HUD

高度：24-28 px。

内容：

- 左：`Ver.5`
- 中：宠物名，例如 `Zurumon`
- 右：`Q1-1` 或 `0h`

样式：

- 字号 12-14。
- 不加外框。
- 不显示 `DIGITAL MONSTER` 品牌条。
- 不显示 `A` / `B` 键。
- 不要把标题做成大号高亮，宠物才是主角。

### 3.2 宠物 LCD 舞台

高度：约 400 px。

内容：

- 宠物 sprite 居中偏下。
- 只保留一条简单地面线。
- 默认不显示 skyline 方块。
- 没有 CALL/病/睡眠/冷冻时，不显示额外标识。

边框：

- 默认只允许一层 LCD 边框。
- 边框 1-2 px 即可。
- 不要使用当前截图里的粗亮绿色大边框。
- 不要在 LCD 内再套一层状态栏边框。

宠物大小：

- Baby / Child 阶段 sprite 视觉尺寸建议 96-128 px。
- 当前截图里宠物太小，至少放大 20-30%。
- 保持 `image-rendering: pixelated` 或 Vela 等效硬边显示。

宠物位置：

- X 居中。
- Y 位于舞台中下部，不贴底。
- 排泄物、伤病图标出现在宠物附近，不能出现在远离宠物的 UI 区。

### 3.3 进化倒计时

`I EVO 6m` 不应占一整条横栏。

改为：

- 放在顶部 HUD 右侧，形式：`EVO 6m`
- 或在舞台左上角小字显示。

样式：

- 字号 11-12。
- 弱绿色。
- 不加边框，不独占横条。

### 3.4 HUN / POW

默认首页仍可显示心数，但要更克制。

推荐一行：

```text
HUN ####     POW ####
```

样式：

- 高度 28-36 px。
- 不使用两个大矩形盒子。
- 不加粗边框。
- 字号 13-15。
- 使用弱绿或灰绿，满心 `####` 可略亮。

如果要保留盒子：

- 边框 1 px。
- 高度不超过 28 px。
- 背景和页面接近，不要像按钮。

### 3.5 TAP 提示

默认首页不显示 `MENU` 按钮。

允许显示很弱的：

```text
TAP
```

或：

```text
TAP SCREEN
```

要求：

- 字号 10-12。
- 颜色非常弱，例如深绿。
- 不加边框。
- 不像按钮。
- 可以 2 秒后淡出，或常驻低亮。

## 4. 默认首页禁止项

默认首页禁止出现：

- `MENU` 按钮。
- 9 宫格或 3 按钮常驻菜单。
- 大号 `DIGITAL MONSTER` 品牌条。
- `A` / `B` 虚拟按键装饰。
- 多层边框。
- 大面积按钮区。
- 大号消息条。
- 长文本说明。
- skyline 方块抢占主视觉。
- 状态页式数据列表。

首页只表达一件事：宠物正在这里。

## 5. 点屏后的菜单状态

### 5.1 菜单打开方式

触发：

- 点按任意非按钮区域。

表现：

- 底部出现半屏以内的命令层。
- 宠物舞台仍可见。
- 不跳到独立菜单页。

菜单区域高度：

- 96-128 px。
- 从底部浮出或直接出现。
- 不需要复杂动效，出现即可。

### 5.2 Command 菜单内容

第一层菜单：

```text
FOOD   TRAIN  BATTLE
WC     MED    LIGHT
COLD   STATUS DATA
BACK
```

布局建议：

- 3 列 x 3 行，外加 BACK。
- 或 4 列 x 2 行 + BACK。
- 每个按钮高度 32-36 px。
- 字号 11-12。
- 间距 4-6 px。
- 按钮必须比当前 `MENU` 大按钮弱，不要成为主视觉。

菜单打开时可以显示一条短消息：

```text
CALL / SELECT CARE
```

但消息条高度不超过 24 px。

### 5.3 菜单收起

触发：

- 点 `BACK`。
- 完成一个非子菜单动作。
- 5-8 秒无操作。
- 再次点宠物舞台。

FOOD 子菜单例外：投喂后可以停留，方便连续补心。

## 6. 首页状态叠加

状态叠加应作用在宠物舞台，而不是变成额外 UI 面板。

### 6.1 Normal

- 无 `CALL`。
- 无消息条。
- 宠物待机动画循环。
- 只显示弱 HUD 与 HUN/POW。

### 6.2 Call

- 顶部或舞台角落显示 `CALL`。
- `CALL` 低帧闪烁。
- 点屏打开菜单后，相关动作高亮。

高亮规则：

- 饥饿/力量低：高亮 `FOOD`。
- 排泄物：高亮 `WC`。
- 病/伤：高亮 `MED`。
- 睡眠未关灯：高亮 `LIGHT`。

### 6.3 Poop

- 排泄物直接画在宠物旁边。
- 不需要单独消息条。
- 菜单打开时高亮 `WC`。

### 6.4 Sick / Hurt

- 宠物附近显示小型 `SICK` / `HURT`。
- 或顶部角落短标签。
- 菜单打开时高亮 `MED`。

### 6.5 Sleep

- 舞台整体变暗。
- 宠物静止。
- 显示 `ZZZ`。
- 点屏菜单只显示 `LIGHT/WAKE`、`STATUS`、`BACK` 等必要动作。

### 6.6 Cold

- 舞台变为低亮。
- 宠物静止。
- 显示 `COLD`。
- 点屏菜单只显示 `RESUME`、`STATUS`、`BACK`。
- 不显示 FOOD/TRAIN/BATTLE 等不可用动作。

### 6.7 Dead

- 不显示普通菜单。
- 显示墓碑或数据化符号。
- 只提供 `NEW EGG`，并进入二次确认。

## 7. 待机动画规范

开发必须实现宠物待机动画。

最低要求：

- 2 帧循环。
- 每帧停留 500-800 ms。
- 位置变化 6-12 px。
- 可以左右走一步，也可以轻微上下跳。

不允许：

- 平滑 tween。
- 缩放呼吸。
- 透明度渐隐渐现。
- 模糊或抗锯齿拉伸。

示例节奏：

```text
Frame A: x = center - 6, y = base
Frame B: x = center + 6, y = base - 4
```

如果 sprite 本身有 GIF 帧，优先使用原 sprite 帧；如果 Vela 不适合播放 GIF，则用同一 PNG 做位置跳帧。

## 8. 给开发的首页返工清单

必须删除或隐藏：

- 默认态 `MENU` 按钮。
- 顶部 `A` 小框。
- 顶部 `DIGITAL MONSTER` 品牌条。
- `I EVO 6m` 的整条状态栏。
- 默认态消息条。
- 默认态大按钮区。
- skyline 方块。
- 两个厚边框 HUN/POW 盒。

必须保留或新增：

- 宠物 sprite 放大到 96-128 px。
- 单层细 LCD 边框。
- 极简顶部 HUD。
- 一行式 `HUN ####  POW ####`。
- 弱 `TAP` 提示。
- 点屏唤出底部 Command 菜单。
- 待机低帧动画。
- 状态高亮规则。

## 9. 建议的首页视觉规格

### 9.1 默认态尺寸

```text
.page
  width: 432
  height: 514
  padding: 12-16

.topHud
  height: 24-28

.lcdStage
  height: 390-410
  border: 1-2

.petSprite
  width: 96-128
  height: 96-128

.meters
  height: 28-36

.tapHint
  height: 20-24
```

### 9.2 菜单态尺寸

```text
.commandLayer
  position: bottom
  height: 104-128

.commandButton
  width: 24-31%
  height: 32-36
  margin: 4-6
  border: 1-2
  font-size: 11-12
```

菜单态可以略微压缩舞台或覆盖底部，但不能遮挡宠物主体。

## 10. 开发验收标准

开发交付后用这 10 条检查：

1. 打开首页，第一眼是否只有宠物待机画面，而不是菜单页。
2. 默认态是否没有可见 `MENU` 按钮。
3. 宠物是否是最大视觉焦点。
4. 宠物是否有低帧待机动画。
5. 点屏是否才出现 Command 菜单。
6. 菜单是否在底部，且不遮挡宠物主体。
7. 首页是否只保留一层主要边框。
8. HUN/POW 是否像状态读数，而不是两个大按钮。
9. 普通状态是否没有大消息条。
10. Cold/Sleep/Dead 是否裁剪菜单，而不是照常显示所有动作。

如果任意一条不满足，就还没有达到“足够简化”的首页标准。

## 11. 一句话方向

首页不是“有宠物的控制台”，而是“一只宠物正在手表 LCD 里待机”。菜单只是用户点屏后临时出现的照顾工具。
